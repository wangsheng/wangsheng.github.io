环境准备

  1. 使用Github账号授权Travis-CI

    使用Github账号登录Travis-CI，并授权权限以及要进行CI持续化集成的版本库。

    注意：为了防止后面，travis-ci会根据默认的配置，构建master分支，这里需要勾选中第一个：

  2. 进阶上一篇博客Github+Hexo+Markdown写博客, 创建blog分支，用于存放博客站点源码以及markdown原始文件

  3. 创建travis-ci构建需要的配置文件

  4. 生成秘钥

    在生成过程中,passphrase 留空，因为travis-ci 中输入密码无法做到。同时使用默认的保存目录就好(~/.ssh/id_rsa)。

  5. 添加生成好的 SSH key 到 ssh-agent

  6. 将公钥配置到 github 上项目的 deploy key 中

    注意：这里需要选中 Allow write access

  7. 将仓库的远程源改为ssh协议，避免https每次让输入用户账户信息

  8. 将_config.yml中deploy小节中的地址也改为ssh协议

travis-ci命令安装与准备

  1. 安装travis命令行工具

    本地安装 travis-ci。（travis 工具采用 ruby开发。需要本地安装有 ruby 开发环境）
